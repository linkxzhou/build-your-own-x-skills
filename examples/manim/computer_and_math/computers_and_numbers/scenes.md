# 计算机与数字：二进制的奥秘

## 概述
- **主题**: 计算机如何表示和处理数字
- **Hook**: 为什么 0.1 + 0.2 ≠ 0.3？计算机真的会算错吗？
- **目标受众**: 编程初学者、计算机科学入门者
- **预计时长**: 约 8-10 分钟
- **核心洞见**: 计算机的"思维"基础是二进制，所有数字最终都要翻译成0和1

## 叙事主线
从一个简单的问题开始——为什么计算机会"算错" 0.1 + 0.2？引出计算机只能理解开和关（1和0）的本质。逐步揭示进制、整数存储、浮点数表示的奥秘，最终让观众理解这个"错误"背后的深刻原因。

**时长**: ~90 秒

## Scene 4: 浮点数的存储 - IEEE 754 标准
**时长**: ~120 秒
**目的**: 解释浮点数的存储结构，揭示精度问题的根源

### 内容

3. 类比科学计数法：±1.xxxxx × 2^指数
4. 0.1 的真相：二进制中是无限循环小数！

### 旁白要点
- 揭示开场谜题的答案
- 强调这不是bug，而是有限内存的必然结果

### 技术要点

## 数学内容
- 二进制转十进制: 1011₂ = 1×2³ + 0×2² + 1×2¹ + 1×2⁰ = 11
- 十六进制转十进制: A3₁₆ = 10×16¹ + 3×16⁰ = 163
- n位无符号整数范围: [0, 2ⁿ - 1]
- IEEE 754 单精度: 1位符号 + 8位指数 + 23位尾数
- 浮点数值公式: (-1)^S × 1.M × 2^(E-127)
