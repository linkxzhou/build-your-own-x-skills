## Scene 1: 函数——集合之间的映射 (scene_01_functions.py)

### 1.1 开场与引入
- 标题动画："函数与关系"
- 副标题："从映射到抽象"
- 引出问题：函数不只是 y = f(x)，它是一种更一般的概念

### 1.2 函数的严格定义
- 两个集合 S（定义域）和 T（值域）
- 函数 f: S → T 是有序对 (s, t) 的集合
- 关键约束：S 中每个元素必须出现一次且仅出现一次作为第一元素
- 动画：展示配对规则

### 1.3 函数的可视化（箭头图）
- 用两个椭圆表示集合 S 和 T
- 用箭头表示映射关系
- 示例：S = {a, b, c}，T = {1, 2, 3}
- 展示合法映射和非法映射的对比

### 1.4 关键理解
- S 中不同元素可以对应 T 中同一元素（允许）
- S 中一个元素不能对应 T 中多个元素（禁止）
- 类比："每人分配一个座位"

## Scene 2: 函数的特殊性质 (scene_02_properties.py)

### 2.1 满射（Onto/Surjective）
- 定义：T 中每个元素都被至少一个 S 元素指向
- 可视化：T 中所有元素都有箭头指向
- 类比："所有座位都坐满了"

### 2.2 单射（One-to-One/Injective）
- 定义：S 中不同元素对应 T 中不同元素
- 可视化：没有两个箭头指向同一个 T 元素
- 类比："没有两人坐同一座位"

### 2.3 双射（Bijection）
- 定义：既是满射又是单射
- 可视化：完美的一一对应
- 特性：双射函数存在反函数
- 示例：字母与数字的一一对应

### 2.4 对比总结
- 三种性质的对比图
- 元素数量关系：|S| vs |T|

## Scene 3: 函数复合 (scene_03_composition.py)

### 3.1 复合定义
- 两个函数 f: S → T 和 g: T → U
- 复合函数 (g ∘ f): S → U
- 公式：(g ∘ f)(s) = g(f(s))

### 3.2 可视化
- 三个集合 S, T, U
- 先 f 后 g 的"接力"过程
- 动画展示：s → f(s) → g(f(s))

### 3.3 性质
- 复合顺序重要：g ∘ f ≠ f ∘ g（一般情况）
- 结合律成立：(h ∘ g) ∘ f = h ∘ (g ∘ f)
- 编程类比：函数调用链

## Scene 4: 二元关系——对函数的抽象 (scene_04_relations.py)

### 4.1 从函数到关系
- 放松函数的严格限制
- 二元关系更"自由"

### 4.2 关系的定义
- 给定两个集合 A 和 B
- 笛卡尔积 A × B = 所有有序对 (a, b) 的集合
- 二元关系 R 是 A × B 的任意子集
- 与函数的区别：
  - A 中元素可以对应 B 中零个、一个或多个元素
  - A 中某些元素可以不参与配对

### 4.3 关系的三种表示
#### 4.3.1 集合列举法
- 直接列出所有有序对
- 示例：R = {(1,a), (1,b), (2,b)}

#### 4.3.2 矩阵表示法
- 行代表 A 元素，列代表 B 元素
- 1 表示有关系，0 表示无关系
- 动画：构建关系矩阵

#### 4.3.3 有向图表示法
- 点代表元素
- 箭头代表关系
- 动画：绘制有向图

### 4.4 关系的计数
- 两个有限集合 A 和 B
- 可能的关系数量：2^(|A| × |B|)
- 示例：|A|=2, |B|=2 → 16种关系

## Scene 5: 关系的运算 (scene_05_operations.py)

### 5.1 关系的复合
- R: A → B 和 S: B → C
- 复合 S ∘ R: A → C
- 规则：(a, c) ∈ S ∘ R ⟺ ∃b: (a,b)∈R ∧ (b,c)∈S
- 矩阵表示：布尔矩阵乘法

### 5.2 逆关系
- R^(-1)：把所有有序对方向调换
- 如果 (a, b) ∈ R，则 (b, a) ∈ R^(-1)
- 矩阵表示：转置

### 5.3 补关系
- 所有不在 R 中的 A × B 有序对
- 矩阵表示：0和1互换

### 5.4 N元关系与数据库
- 从二元推广到 n 元
- n 元组 (a₁, a₂, ..., aₙ)
- 数据库表就是 n 元关系

## Scene 6: 等价关系 (scene_06_equivalence.py)

### 6.1 定义
- 在集合 S 上的二元关系 R
- 三个性质：
  - 自反性：∀s: (s, s) ∈ R
  - 对称性：(a, b) ∈ R → (b, a) ∈ R
  - 传递性：(a, b) ∈ R ∧ (b, c) ∈ R → (a, c) ∈ R

### 6.2 经典例子
- 整数相等关系
- 三角形全等/相似关系
- 模 n 同余关系（除以 n 余数相同）

### 6.3 等价类
- 等价关系把集合划分成互不相交的子集
- 同一等价类中的元素都是"等价"的
- 示例：模3同余把整数分成三类（余0、余1、余2）

### 6.4 用等价关系构建数集
- 从自然数对构建整数：(a, b) 代表 a - b
- 从整数对构建有理数：(m, n) 代表 m/n
- 体现数学的构造性思维

## Scene 7: 偏序关系 (scene_07_partial_order.py)

### 7.1 定义
- 在集合 S 上的二元关系（记作 ≤）
- 三个性质：
  - 自反性：∀s: s ≤ s
  - 反对称性：a ≤ b ∧ b ≤ a → a = b
  - 传递性：a ≤ b ∧ b ≤ c → a ≤ c

### 7.2 与等价关系的对比
- 等价关系：对称性（双向）
- 偏序关系：反对称性（单向或相等）

### 7.3 偏序的例子
- 集合的包含关系 ⊆
- 整数的小于等于 ≤
- 任务的依赖关系

### 7.4 偏序集
- 配备了偏序关系的集合
- 部分元素可比较，部分不可比较
- "偏"的含义：不是所有元素都能比大小

### 7.5 哈斯图（Hasse Diagram）
- 有限偏序集的可视化
- 简化规则：
  - 省略自反性的环
  - 省略传递性必然存在的边
  - 只保留"覆盖"关系
  - 较小元素在下方
- 示例：{1,2,3}的幂集上的包含关系

## Scene 8: 总结与启示 (scene_08_summary.py)

### 8.1 知识体系回顾
- 函数 → 关系 → 等价关系/偏序关系
- 从具体到抽象的思维方式

### 8.2 计算机科学的应用
- 程序中的函数：输入到输出的映射
- 数据库的表：n 元关系

### 8.3 三个核心启示
1. 抽象的威力：一套理论，多种应用
2. 结构的重要性：识别数据间的隐藏结构
3. 从函数到关系：放松约束，获得更大灵活性
