# Prompt 模板与测试提示词

## build-your-own-x skills 使用模板

当前仓库已保留 `263` 个 build-your-own-x 相关 skill，并为每个 skill 生成了一个 example。示例入口见 [`examples/README.md`](./examples/README.md)，转换计划见 [`plan/README.md`](./plan/README.md)。

### 通用学习 Prompt

```text
[使用Skills: <skill-name>]
我想从零实现「<项目名称>」。
请先检查当前目录是否为空，然后给我一个 4-8 个里程碑的学习计划。
从第一个最小可运行版本开始，只实现第一阶段，并提供验证命令。
```

### 通用调试 Prompt

```text
[使用Skills: <skill-name>]
我已经做到一半，但测试失败了。
请先阅读项目文件，定位当前处于哪个里程碑，再做最小修复并解释原因。
```

### 通用教程转换 Prompt

```text
[使用Skills: <skill-name>]
请基于原始教程资源，把它整理成 Claude Code 可执行的学习任务清单。
每个任务都要包含交付物、关键概念、待修改文件和验收命令。
```

### 具体示例

```text
[使用Skills: database-redis]
我想用 C++ 从零实现「从零构建你自己的 Redis」。

```


```text

我想从零实现一个 Web Server。请先规划 HTTP 请求解析、响应生成、连接处理和测试方式。
```

## examples 生成规则

- examples 按大类型分组，如 `examples/database/`、`examples/programming-language/`。
- 每个 skill 对应一个 `examples/<category>/<skill-name>/README.md`。
- 每个 example 包含推荐 Prompt、预期交互、学习目标、原始资源和变体测试 Prompt。

```
按照如下内容生成视频，内容中每个 scene 按照知识点展开，尽可能用详细的动画介绍，内容如下：  

<content>
数论，用最简单的话说，就是研究整数（比如 -3, 0, 1, 5, 100）的“脾气”和“规律”的数学。虽然它听起来很高深，但却是计算机科学的基石，尤其是在密码学（比如保护你的网络聊天信息）、哈希函数（比如快速定位数据）和设计高效算法中无处不在。
这一章节就像一次寻宝之旅，带你探索整数世界里的各种奇妙现象。我们从最核心的“质数”开始，一步步深入到更复杂的结构。

第一部分：一切的起点——质数
质数是数论的“明星”。它们是那些大于1，并且只能被1和它自身整除的整数（比如2, 3, 5, 7）。不是质数的数叫合数（比如4, 6, 8, 9）。

为什么“1”不算质数？ 这是为了保证一个非常重要的定理——算术基本定理：任何一个大于1的整数，都可以唯一地写成一系列质数相乘的形式。如果1算质数，那这种“唯一性”就被破坏了（比如 6 = 2 × 3，也可以写成 1 × 2 × 3，1 × 1 × 2 × 3… 就没完没了了）。

寻找质数：“筛子”的艺术

暴力法：对于一个数n，检查所有比它小的数是否能整除它。这种方法非常慢，时间复杂度是O(n²)，只适合很小的数。
埃拉托斯特尼筛法：一个古老而高效的方法。

列出从2开始的所有自然数。
圈出第一个数2（它是质数），然后划掉所有2的倍数（4,6,8...）。
下一个没被划掉的数是3（它是质数），划掉所有3的倍数（6,9,12...此时6已被划掉，是重复操作）。
重复这个过程，就像用筛子筛掉合数，留下的就是质数。

质数的“大家族”：

梅森质数：形如 2^p - 1 的质数，其中 p 本身也是质数。比如 2² - 1 = 3， 2³ - 1 = 7。目前已知的最大质数基本都是梅森质数。
孪生质数：相差为2的一对质数，如 (3,5), (5,7), (11,13)。是否存在无穷多对孪生质数，至今仍是未解之谜。
索菲·热尔曼质数：对于一个质数p，如果 2p + 1 也是质数，那么p就是一个索菲·热尔曼质数。

质数测试：判断一个大数是否是质数，有比暴力枚举快得多的算法，比如米勒-拉宾素性测试。它是一种概率算法，通过多次随机测试，能以极高的概率判断一个数是否为质数。

哥德巴赫猜想（未解决！）：任何一个大于2的偶数，都可以写成两个质数之和。例如 4=2+2, 10=3+7, 42=5+37 等等。计算机已经验证到非常大的数字都成立，但严格的数学证明至今没人完成。

第二部分：整除性与最大公约数
“a整除b”（写作 a|b）意味着 b ÷ a 的余数为0。这是研究整数关系的基础。

最大公约数：两个整数a和b的最大公约数（GCD）是能同时整除a和b的最大正整数。例如 GCD(12, 18) = 6。

更相减损术 (欧几里得算法)：计算GCD的高效方法。原理很简单：两个数的最大公约数，等于其中较小的数和两数之差的最大公约数。反复用此原理，直到两数相等，那个数就是GCD。

例：求 GCD(102, 66)
GCD(102, 66) = GCD(66, 102-66=36) = GCD(36, 66-36=30) = GCD(30, 36-30=6) = GCD(6, 30-6=24) … 最终会得到 GCD(6,6)=6。

扩展欧几里得算法：这个算法的升级版不仅能求出GCD，还能找到两个整数s和t，使得 s*a + t*b = GCD(a, b)。这个等式在解某些方程和密码学中至关重要。

最小公倍数：两个整数a和b的最小公倍数（LCM）是能同时被a和b整除的最小正整数。它与GCD有一个漂亮的关系：LCM(a, b) × GCD(a, b) = a × b。

第三部分：模运算——时钟的数学
模运算是数论中极其强大的工具，它关注的是余数。

定义：a ≡ b (mod n)（读作“a与b模n同余”），意思是 a 和 b 除以 n 得到的余数相同。这等价于 n 能整除 (a - b)。

例：17 ≡ 5 (mod 6)，因为 17÷6 余5，5÷6 也余5。同时，17-5=12，12能被6整除。

直观理解：把它想象成一个只有n个刻度的钟（比如12小时制的钟）。在这个钟上，13点和1点指向的是同一个位置，所以 13 ≡ 1 (mod 12)。

模线性方程：求解形如 ax ≡ b (mod n) 的方程。它有解的条件是 GCD(a, n) 能整除 b。利用扩展欧几里得算法可以找到解。

第四部分：丢番图方程——寻找整数解
丢番图方程是指只关心整数解的方程。

线性丢番图方程：形如 ax + by = c（a, b, c是整数）。它有整数解的条件同样是 GCD(a, b) 能整除 c。
毕达哥拉斯三元组：满足 a² + b² = c² 的正整数解，即直角三角形的整数边，如 (3,4,5), (5,12,13)。
费马大定理（已解决！）：方程 a^n + b^n = c^n 在 n > 2 时没有正整数解。这个看似简单的结论，却耗费了数学家358年的时间才被最终证明。
埃尔德什-施特劳斯猜想（未解决！）：猜想对于任何 n ≥ 2，方程 4/n = 1/a + 1/b + 1/c 都有正整数解。目前对极大的n都验证成立，但尚未被普遍证明。

第五部分：奇妙的整数序列
数论学家收集和研究各种有趣的整数序列，就像集邮一样。

因数和函数：

σ(n)：表示n的所有正因数之和（包括1和n本身）。
真因数和 s(n)：σ(n) - n，即除n本身以外的因数之和。

完全数、亲和数与相亲数链：

完全数：一个数等于它的真因数和。例如 6 = 1+2+3， 28 = 1+2+4+7+14。
亲和数：两个数 (a, b)，满足 a 的真因数和等于 b，并且 b 的真因数和等于 a。例如 (220, 284)。
相亲数链：一串数，其中每个数的真因数和都等于下一个数，最终形成一个环。

欧拉φ函数 φ(n)：计算在 1 到 n 之间，与 n 互质（最大公约数为1）的数的个数。在RSA公钥加密算法中扮演核心角色。
哈沙德数：一个数能被它各位数字之和整除。例如 18 ÷ (1+8)=2，所以18是哈沙德数。
错位排列数（子阶乘） !n：求把n个元素重新排列，使得没有任何一个在原来位置的方法有多少种。例如3个字母ABC，错位排列有2种：BCA, CAB。
史密斯数：一个合数，其各位数字之和，等于它的所有质因数各位数字之和。例如 4937775 → 4+9+3+7+7+7+5=42，它的质因数 3,5,5,6,5,8,3,7 之和也是42。
鲁斯-阿伦数：两个连续的正整数（如714和715），它们的质因数之和相等。

总结与启示

数学的纯粹之美：数论研究的问题往往非常简单直观（只涉及整数），但其深度和难度却超乎想象，充满了未解之谜，体现了数学的纯粹魅力。
计算机科学的核心工具：从加密你的网络支付（RSA算法），到设计高效的哈希表和随机数生成器，再到算法分析，数论的概念和工具是现代计算不可或缺的一部分。例如，计算机的有限精度运算本质上就是一种模运算。
抽象与具体的桥梁：它从最具体、最熟悉的整数出发，发展出一套高度抽象和严谨的理论（如同余理论），并用这些理论来解决极为实际的安全和效率问题。
探索的乐趣：数论就像一座宝藏，既有像哥德巴赫猜想这样宏伟的未解之谜等待攻克，也有像亲和数、史密斯数这样小巧玲珑的“宝石”供人欣赏把玩。

总而言之，《数论》这一章为你打开了整数世界的大门。它告诉你，即使是最基础的整数，也蕴藏着深不可测的规律、无与伦比的美感和极其强大的应用价值。 学习数论，不仅是学习一套数学工具，更是接受一次逻辑与想象力的绝佳训练。
</content>
```

```
<content>

《给高中生的深度学习入门手册》
你好，未来的AI创造者！
你可能已经用过手机上的语音助手、见过能生成逼真图片的AI，或者听说过可以和你对话的智能机器人。这些酷炫技术的核心，就是深度学习。别被这个名字吓到，它其实就像教一个“机器大脑”学习，而这本书就是最好的手把手教程。
为了让知识更系统，我们将完全跟着一本经典小书《The Little Book of Deep Learning》的目录来探索。准备好了吗？我们出发！

第一部分：地基——机器学习究竟是什么？
想象一下，你要训练一只小狗“坐下”。你不会给它讲道理，而是先演示几次“坐下”的动作，它做对了就给零食（奖励），做错了就没有（惩罚）。训练多了，它就能学会。
深度学习 就是一种高级的“训练机器”的方法。

核心任务：我们给机器看很多“例子”（比如成千上万张猫和狗的图片，并告诉它哪张是猫），让它自己总结规律。最终，面对一张它没见过的猫图，它也能认出来。

训练集：我们给机器学习的那些“例子”。
模型：机器用来学习的那个“数学公式”或“程序规则”。
参数：模型内部可以调整的“旋钮”，就像前面训练小狗时零食的大小——调得好，学得快。
损失：衡量机器当前“猜得有多差”的分数。我们的目标就是通过各种方法，把这个分数降到最低。

不过，这里有个大坑：过拟合
想象一个学生，为了应付数学考试，把《五年高考三年模拟》的每道题答案都死记硬背下来。这看起来很棒，但一旦考试出了一道新题型，他就傻眼了。因为他只“记住”了题目，没有真正“理解”知识。
这就是过拟合：模型在训练集上表现完美，但对新数据一塌糊涂。所以，我们必须要把数据分成三份：

训练集：让学生学习的课本和习题。
验证集：用来测试他学习效果，并帮他调整学习方法（比如是题海战术还是理解概念）的模拟考卷。
测试集：最终决定他真实水平的“高考”试卷，在最终评估前绝对不能让他看到。

克服过拟合需要一些技巧（比如下一部分会讲的Dropout），这体现了深度学习的一个核心思想：优秀的模型不仅需要强大的计算能力，更需要巧妙的设计来防止它“学傻了”。

第二部分：引擎房——深度学习的“硬核”计算
要训练一个强大的“机器大脑”，需要海量的计算，这离不开强大的硬件。

GPU（图形处理器）：你电脑里用来打游戏、让画面超级流畅的芯片。处理图像需要同时进行大量简单计算，这个特性恰好被深度学习“借用”来做矩阵运算。可以说，游戏产业的繁荣，意外地为AI革命提供了燃料。
TPU（张量处理器）：谷歌公司专门为深度学习设计的“定制芯片”，就像从“瑞士军刀”（GPU）升级到了“专业厨刀”（TPU），效率更高。
张量：这是深度学习中数据的基本单位。别怕这个词！

标量：一个数字，比如温度 36.5。
向量：一组数字，比如你的身高体重 [175, 65]。
矩阵：一个表格数字，比如全班同学的成绩表。
张量：可以理解为更高维度的矩阵。一张RGB图片，就可以看作一个 3 x 高 x 宽 的张量（红、绿、蓝三个通道）。

理解了数据（张量）和算力（GPU），接下来我们看最重要的环节：训练。

第三部分：训练室——“机器大脑”是如何学习的
步骤1：定义“好坏”——损失函数
就像考试需要“分数”来衡量表现一样，我们需要一个函数来告诉模型：“你这个预测离正确答案还差多远”。常见的“评分标准”有：

均方误差（MSE）：常用于预测房价、分数等数值。
交叉熵损失：常用于图片分类（这是猫还是狗？）。

步骤2：反向传播——“闻着味儿”找错误来源
假设你在一个全黑的迷宫里，想走到出口（损失最低点）。你的脚能感觉到脚下的坡度（梯度），知道哪个方向是“下坡”方向。
梯度下降 就是沿着最陡的“下坡”方向，小心翼翼地走一小步。在AI中，这“一步”的大小叫 学习率，是训练中最关键的“开关”之一。
那怎么知道整个迷宫哪里的坡度最陡呢？ 这就要靠 反向传播，它就像是从出口处的错误，一层层倒推，精确算出迷宫中每个岔路口对最终走错路该负多少责任。这个算法是深度学习的基石，让海量参数的高效调整成为可能。
但训练深度模型时，一个“致命”问题出现了：梯度消失/爆炸。在反向传播时，梯度信号经过很多层后会变得极其微弱（消失）或巨大（爆炸），导致前面层的参数几乎学不到东西。这引出了本书的一个重要观点：与其只改进优化算法，不如把模型本身设计得更容易被优化。这催生了ResNet、Transformer等划时代的架构。
步骤3：“深度”的价值——多层协作
为什么模型要“深”（有很多层）？看一个简单例子：用8层小网络对两类点做分类。

一开始，两类点杂乱地混在一起。
训练中，每一层都对数据进行一点微小的“扭曲”。
经过多层协作后，数据被扭曲成两类点可以被一条直线轻易分开的模样，最后一层只需“画条线”即可完成分类。

这模拟了深度网络的功能：通过一系列变换，将原始、杂乱的数据，一步步变成更容易被处理和分析的表示形式。在现实中，这些表示发生在成百上千维的空间里，为我们处理图像、声音和文字提供了可能。

第四部分：积木库——组成AI模型的“乐高”零件
现代深度学习模型是由标准化的“层”像搭乐高一样堆叠而成的。
1. 线性层（全连接层）
最基本的组件。想象一个神经元，它接收一组输入信号，给每个信号赋予一个“权重”（重要性），最后加总并加上一个“偏置”，产生输出。一层就是由大量这样的神经元组成。
2. 卷积层
这是处理图像和声音的利器！它像一个“局部放大镜”，每次只看图像的一小块区域（比如3x3像素），提取局部特征（如边缘、纹理）。通过在整张图上滑动这个放大镜，它能高效地捕获空间信息。
3. 激活函数——引入“非线性”
如果只有线性层叠加，无论堆多少层，整体还是一个线性变换，能力有限。激活函数就是给模型加入“弯弯绕”的“香料”。

ReLU（整流线性单元）：最简单的激活函数。把所有负数变为0，正数则保留。就像一个开关，过滤掉了“负能量”信号。
在这个“开关”基础上，后来还发展出了Leaky ReLU、GELU等变体，解决了ReLU的一些小问题。

4. 池化层
主要用来“压缩”数据。比如最大池化，在一个小区域（如2x2网格）里只保留数值最大的那个，舍弃其他。这能减少计算量，并使模型更关注某个特征“是否出现”，而非它“精确出现在哪”。
5. Dropout层——防止“死记硬背”
训练时，这一层会随机地将一部分神经元的输出强制设为0。这听起来很暴力，但效果神奇：它强迫网络不能过分依赖任何少数几个神经元，必须学会用不同的“团队”来分散学习特征，就像你不能只靠一个知识点来应对所有考题。这大大增强了模型的泛化能力，是防止“过拟合”的利器。
6. 归一化层——让训练更稳定
训练时，网络前面层的参数微小变化，经过多层放大后，可能导致后面层的输入数据分布剧烈波动，让后面层“无所适从”，学习困难。

批归一化：就是对每一批训练数据，进行“标准化”处理（减均值，除以标准差），让输入后续层的数据保持稳定分布，极大加速了训练。

7. 跳跃连接 —— 给梯度开“绿色通道”
这是解决“梯度消失”问题的天才设计！它允许数据直接从网络的某一层，“跳跃”到更后面的层。这样，即使中间某些层的学习信号很弱，梯度也能沿着这条“高速公路”直接传回去，确保前面层也能得到有效更新。著名的ResNet（残差网络） 就是通过这个思想，成功训练了上百层的超深网络。
8. 注意力层 —— 让模型学会“划重点”
这是当前最火的大语言模型（如GPT）的核心！它让模型在处理一个词（或一个图像块）时，可以动态地“关注”序列中任何其他相关的部分。

就像一个学生读历史书时，看到“秦始皇”，会立刻关联到“统一六国”、“焚书坑儒”等内容。
技术实现上，它会为每个位置计算一个查询（Q），然后与所有位置的键（K） 去匹配，得到一个“相关性”分数，再用这个分数对值（V） 进行加权求和。这就是著名的 Q， K， V 自注意力机制。

第五部分：经典架构——明星AI模型的“设计图纸”
了解了基本积木，来看看那些著名的模型是怎么搭建的。
1. 多层感知机（MLP）
最古老的深度架构，就是“线性层 + 激活函数”反复叠加。简单、通用，可以作为复杂模型中的一部分。
2. 卷积神经网络（CNN）
专门为图像处理设计的“王者”。

LeNet：早期的CNN，用于手写数字识别。
ResNet（残差网络）：引入了跳跃连接的里程碑式架构。它的核心是残差块，让网络的目标不再是直接学习一个复杂的映射，而是学习输入与输出之间的“残差”（差值），这使得训练上百层的网络成为可能。

3. Transformer模型
自然语言处理领域的革命者，现在也横扫了图像、音频等领域。核心就是自注意力层和前馈网络（一个MLP）的叠加，并配合前面提到的跳跃连接和层归一化。我们熟知的GPT系列、 ViT（用Transformer做图像分类）都基于此。

第六部分：大展身手——深度学习在做什么？
学了一身本领，AI模型们具体在做什么工作呢？主要分两大类：预测 和 合成。
预测类任务（给你输入，让它猜答案）

图像分类：看图说话，判断图片里是猫是狗。使用CNN或ViT。
目标检测：不仅知道是什么，还要知道在哪。在图上用“框”标出物体位置。YOLO、SSD就是做这个的。
语义分割：更精细，给图片每一个像素都打上标签（这是天空，这是马路，这是行人）。就像给你的照片做了个“分区填色”。
语音识别：把你的声音变成文字。
强化学习：让AI通过不断试错来学习最佳策略，就像训练一只小鼠走迷宫找奶酪。著名的AlphaGo就用了这个。

合成类任务（让它创造新东西）

文本生成：这就是GPT家族的绝活。你给个开头，它能续写出故事、代码、邮件。关键是自回归：每次只预测下一个词，然后把这个词作为新输入的一部分，再预测下一个，如此循环。
图像生成：如DALL-E、Stable Diffusion。它们通过学习海量图片的分布，可以“凭空”生成一张符合你文字描述的、全新的、高质量的图片。背后常用扩散模型技术，逐步将随机噪声“去噪”成一张清晰图片。

第七部分：挑战与未来——模型的极限与我们的巧思
随着模型变得巨大（如GPT-3有1750亿参数），训练和运行它们需要天价的金钱和电力。如何让普通人也能用上大模型？

提示工程：通过精心设计输入的问题或指令，引导大模型给出更好的回答，而不需要修改模型本身（即零样本或少样本学习）。例如， 思维链技巧，让模型“一步一步地思考”，显著提高了复杂推理的准确性。
量化：将模型参数的精度从32位浮点数降低到8位甚至4位整数。就像一个厨师能用专业天平，也能用小台秤做出好菜。这能大幅减少模型体积和计算开销。
适配器：在已经训练好的大模型上，只新增和训练一小部分参数（适配器），就能让它适应新的特定任务，极大降低了微调成本。
模型合并：一个有趣的研究方向，尝试将多个不同专家模型的“知识”安全地融合到一个模型里。

结语
看，深度学习并不神秘！它就像是：
用数据和计算力（GPU），通过巧妙的算法（梯度下降+反向传播），去优化一个由标准组件（层）精心组装成的深度模型，让它学会预测或创造。
```
